Part 5 • Fungsi: Konsep & Operasi

📖 Pendahuluan

Di bab-bab sebelumnya, kita selalu berurusan dengan persamaan dan pertidaksamaan yang mencari nilai x yang memenuhi suatu kondisi. Itu penting, tapi ada pertanyaan yang lebih dalam: bagaimana suatu besaran bergantung pada besaran lain?

Jawabannya ada pada fungsi—salah satu konsep paling fundamental dalam seluruh matematika. Fungsi adalah cara matematis untuk menyatakan hubungan ketergantungan antara dua variabel.

Setiap kali kamu menekan tombol di kalkulator, kamu menggunakan fungsi. Input 9, tekan √, output 3. Input 100, tekan log, output 2. Fisika penuh dengan fungsi: jarak sebagai fungsi waktu, gaya sebagai fungsi percepatan. Ekonomi juga: harga sebagai fungsi permintaan, pertumbuhan sebagai fungsi waktu.

Di bab ini, kita akan membangun fondasi fungsi secara mendalam: apa definisi tepatnya, apa saja bagian-bagiannya (domain, kodomain, range), bagaimana menggabungkan fungsi (komposisi dan invers), dan bagaimana menangani fungsi yang didefinisikan secara potongan (piecewise).

Tujuan Bab
  • Memahami definisi tepat fungsi dan membedakannya dari relasi biasa.
  • Mampu menentukan domain, kodomain, dan range suatu fungsi dengan berbagai metode.
  • Memahami dan mampu bekerja dengan fungsi piecewise (fungsi potongan).
  • Menguasai operasi komposisi fungsi, termasuk menentukan domainnya.
  • Menguasai konsep invers fungsi: kapan ada, cara mencari, dan verifikasinya.
  • Memahami hubungan antara fungsi dan inversnya secara grafik.

1. Apa Itu Fungsi? — Definisi yang Tepat

Secara formal, fungsi (atau pemetaan) dari himpunan A ke himpunan B adalah aturan yang memasangkan setiap elemen di A dengan tepat satu elemen di B.

f : A → B

A disebut domain (daerah asal)
B disebut kodomain (daerah kawan)

Untuk setiap x ∈ A, f(x) ∈ B adalah peta atau bayangan dari x.

Dua syarat mutlak suatu relasi disebut fungsi:

  1. Setiap elemen di domain harus punya pasangan (tidak boleh ada yang "jomblo").
  2. Setiap elemen di domain punya tepat satu pasangan (tidak boleh "cabang").

Mengapa syarat ini penting?

Bayangkan fungsi sebagai mesin penjual otomatis. Kamu memasukkan kode A1 (input), mesin harus mengeluarkan satu jenis barang (output). Kalau mesin kadang mengeluarkan keripik, kadang cokelat untuk kode yang sama, mesin itu rusak—tidak bisa diandalkan. Begitu juga fungsi: satu input harus menghasilkan satu output yang pasti, agar kita bisa memprediksi hasilnya.

Notasi Fungsi

Notasi f(x) diperkenalkan oleh Leonhard Euler pada abad ke-18.

f(x) = x² + 1

f : nama fungsi (bisa juga g, h, F, G, φ, dll.)
(x) : variabel input
x² + 1 : aturan fungsi ("kuadratkan input, lalu tambah 1")

f(3) artinya: masukkan 3 ke aturan → 3² + 1 = 10.
f(-2) = (-2)² + 1 = 5.
f(0) = 0² + 1 = 1.

Fungsi vs Relasi Biasa: Uji Garis Vertikal

Tidak semua relasi adalah fungsi. Cara termudah mengecek dari grafik: Uji Garis Vertikal.

Jika ada garis vertikal yang memotong grafik di lebih dari satu titik, maka grafik tersebut BUKAN grafik fungsi.

Mengapa? Garis vertikal x = a memotong grafik di titik-titik (a, y₁), (a, y₂), ... Itu berarti input a menghasilkan lebih dari satu output. Melanggar syarat fungsi.

Fungsi: y = x². Garis vertikal x = 2 hanya memotong di (2, 4).

Bukan fungsi: x = y² (atau y = ±√x). Garis vertikal x = 4 memotong di (4, 2) dan (4, -2).

Menghitung Nilai Fungsi

Ini adalah keterampilan paling dasar: mengganti variabel dengan nilai yang diberikan.

f(x) = 3x² - 2x + 5

f(2) = 3(4) - 2(2) + 5 = 12 - 4 + 5 = 13

f(-1) = 3(1) - 2(-1) + 5 = 3 + 2 + 5 = 10

f(0) = 3(0) - 2(0) + 5 = 5

f(a) = 3a² - 2a + 5 (hanya ganti simbol, aturan tetap)

f(x+1) = 3(x+1)² - 2(x+1) + 5
= 3(x² + 2x + 1) - 2x - 2 + 5
= 3x² + 6x + 3 - 2x - 2 + 5
= 3x² + 4x + 6

2. Domain — Semua Input yang Diperbolehkan

Domain adalah himpunan semua nilai x yang membuat fungsi terdefinisi (menghasilkan output real yang valid).

Saat menentukan domain, kita bertanya: "Apa yang bisa salah?" Ada tiga "musuh" utama yang membuat fungsi tidak terdefinisi:

1. Pembagian dengan NOL → penyebut ≠ 0
2. Akar genap dari bilangan NEGATIF → radikan ≥ 0
3. Logaritma dari bilangan NEGATIF atau NOL → numerus > 0

Mengapa pembagian dengan nol tidak terdefinisi?

Bayangkan 6 ÷ 0 = ?. Itu artinya kita mencari bilangan yang kalau dikali 0 menghasilkan 6. Tidak ada bilangan seperti itu. Berapapun dikali 0 hasilnya 0, bukan 6. Jadi pembagian dengan nol tidak memiliki makna dalam bilangan real.

Mengapa akar negatif tidak real? √(-4) = ? Bilangan real berapapun kalau dikuadratkan hasilnya ≥ 0. Tidak ada bilangan real yang kuadratnya -4. Jadi √(-4) bukan bilangan real.

Cara Sistematis Menentukan Domain

  1. Tuliskan semua syarat yang membuat fungsi terdefinisi.
  2. Selesaikan setiap syarat (biasanya berbentuk pertidaksamaan).
  3. Iriskan semua solusi. Itulah domainnya.
  4. Tulis dalam notasi interval atau notasi himpunan.

Contoh 1: Fungsi Polinomial

f(x) = 2x³ - 5x² + 3x - 7

Tidak ada pembagian, tidak ada akar, tidak ada logaritma.
Domain: semua bilangan real, (-∞, ∞) atau ℝ ✅

Fakta penting: Semua fungsi polinomial terdefinisi untuk semua bilangan real. Domainnya selalu ℝ.

Contoh 2: Fungsi Rasional (Ada Pecahan)

f(x) = 5 / (x - 3)

Syarat: penyebut ≠ 0 → x - 3 ≠ 0 → x ≠ 3

Domain: {x | x ≠ 3} atau (-∞, 3) ∪ (3, ∞) ✅

Contoh 3: Fungsi dengan Akar

f(x) = √(2x - 6)

Syarat: radikan ≥ 0 → 2x - 6 ≥ 0 → 2x ≥ 6 → x ≥ 3

Domain: [3, ∞) ✅

Contoh 4: Fungsi dengan Akar di Penyebut

f(x) = 1 / √(x - 2)

Syarat 1: radikan ≥ 0 → x - 2 ≥ 0 → x ≥ 2
Syarat 2: penyebut ≠ 0 → √(x-2) ≠ 0 → x - 2 ≠ 0 → x ≠ 2

Gabungan: x ≥ 2 dan x ≠ 2 → x > 2

Domain: (2, ∞) ✅

Perhatikan: x = 2 membuat penyebut = 0 (karena √0 = 0). Jadi harus dikeluarkan, meskipun radikannya 0.

Contoh 5: Fungsi dengan Dua Syarat

f(x) = √(x+1) / (x-2)

Syarat 1: radikan ≥ 0 → x + 1 ≥ 0 → x ≥ -1
Syarat 2: penyebut ≠ 0 → x - 2 ≠ 0 → x ≠ 2

Gabungan: x ≥ -1 dan x ≠ 2

Domain: [-1, 2) ∪ (2, ∞) ✅

3. Kodomain dan Range — Dua Hal yang Sering Tertukar

Kodomain dan range sama-sama berkaitan dengan output fungsi, tapi keduanya berbeda secara fundamental.

Kodomain: Himpunan tempat output diizinkan berada. Ditentukan oleh pembuat fungsi (bagian dari definisi fungsi).

Range (Daerah Hasil): Himpunan output aktual yang benar-benar dihasilkan ketika semua input dari domain dimasukkan.

Range ⊆ Kodomain (range selalu subset dari kodomain).

Analogi:

Seorang pemanah (fungsi) memanah ke target (kodomain). Targetnya adalah seluruh papan panahan. Tapi anak panah hanya mengenai satu titik di papan itu. Bekas anak panah itulah range.

Kodomain = seluruh papan. Range = titik-titik yang benar-benar terkena panah.

Menentukan Range — Berbagai Metode

Menentukan range lebih menantang dibanding domain. Berikut beberapa metode:

Metode 1: Substitusi dan Observasi (untuk fungsi sederhana)

f(x) = x² (domain: ℝ)

x² selalu ≥ 0 untuk semua x real.
Saat x = 0, f(0) = 0.
Saat x → ∞, f(x) → ∞.

Range: [0, ∞) ✅

Metode 2: Mengubah ke Bentuk y = f(x) dan Mencari Syarat x Real

Ini metode paling sistematis: nyatakan x dalam y, lalu cari syarat agar x real.

f(x) = 1/(x-2)

y = 1/(x-2)
x - 2 = 1/y
x = 2 + 1/y

Agar x real, 1/y harus real. Satu-satunya masalah: y = 0 → 1/0 tidak terdefinisi.

Jadi, y ≠ 0.

Range: (-∞, 0) ∪ (0, ∞) ✅

Metode 3: Mencari Nilai Maksimum/Minimum (untuk fungsi kuadrat)

f(x) = x² + 4x + 1 (domain: ℝ)

Ubah ke bentuk kuadrat sempurna:
f(x) = (x² + 4x + 4) - 4 + 1
= (x + 2)² - 3

(x + 2)² ≥ 0, jadi nilai minimum terjadi saat (x+2)² = 0 → f(-2) = -3.
Saat x → ±∞, f(x) → ∞.

Range: [-3, ∞) ✅

Metode 4: Analisis Perilaku Ujung dan Asimtot

f(x) = (2x+1)/(x-1), x ≠ 1

Bagi pembilang dan penyebut dengan x:
f(x) = (2 + 1/x)/(1 - 1/x)

Saat x → ∞: 1/x → 0, f(x) → 2/1 = 2.
Saat x → -∞: f(x) → 2 juga.
Tapi apakah f(x) pernah tepat = 2?

y = 2 → (2x+1)/(x-1) = 2 → 2x+1 = 2x-2 → 1 = -2 (kontradiksi).
Jadi y ≠ 2.

Range: (-∞, 2) ∪ (2, ∞) ✅

4. Fungsi Potongan (Piecewise Function) — Satu Nama, Banyak Aturan

Fungsi piecewise (fungsi potongan, fungsi sepotong-sepotong) adalah fungsi yang didefinisikan dengan aturan berbeda untuk interval domain yang berbeda.

f(x) = { ekspresi₁, jika syarat₁
        { ekspresi₂, jika syarat₂
        { ekspresi₃, jika syarat₃

Mengapa perlu piecewise?

Dunia nyata penuh dengan situasi di mana aturan berubah tergantung kondisi:

  • Tarif ojol: harga berbeda untuk km pertama vs km berikutnya.
  • Pajak progresif: persentase pajak naik untuk penghasilan di atas threshold tertentu.
  • Nilai mutlak: |x| = x untuk x ≥ 0, tapi -x untuk x < 0.
  • Fungsi tangga (step function): biaya parkir per jam.

Piecewise adalah cara matematis untuk menangani "jika-maka" dalam fungsi.

Contoh 1: Nilai Mutlak sebagai Piecewise

f(x) = |x|

Definisi nilai mutlak:
|x| = x, jika x ≥ 0
|x| = -x, jika x < 0

Jadi:
f(5) = 5 (karena 5 ≥ 0, pakai aturan x)
f(-3) = -(-3) = 3 (karena -3 < 0, pakai aturan -x)
f(0) = 0 (karena 0 ≥ 0, pakai aturan x)

Domain: ℝ (semua bilangan real)
Range: [0, ∞) (nilai mutlak tidak pernah negatif)

Contoh 2: Fungsi Tangga (Tarif Parkir)

Biaya parkir:
- 2 jam pertama: Rp5.000
- Setiap jam berikutnya: Rp2.000 per jam
- Maksimal Rp15.000 per hari

f(t) = { 5000,              jika 0 < t ≤ 2
        { 5000 + 2000⌈t-2⌉, jika 2 < t ≤ 7
        { 15000,             jika t > 7

f(1,5) = 5000
f(4) = 5000 + 2000⌈2⌉ = 5000 + 4000 = 9000
f(10) = 15000 (maksimum)

Contoh 3: Menentukan Domain dan Range Piecewise

f(x) = { x²,   jika -2 ≤ x < 1
        { 2x-1, jika 1 ≤ x ≤ 4

Domain: [-2, 4] (gabungan dari kedua interval).

Range:

Potongan 1 (x², -2 ≤ x < 1):
Saat x = -2: x² = 4
Saat x → 1 (dari kiri): x² → 1
x² pada interval [-2, 1) menghasilkan [0, 4] (karena 0² = 0, dan semua nilai antara 0 dan 4 tercapai).

Potongan 2 (2x-1, 1 ≤ x ≤ 4):
Saat x = 1: 2(1)-1 = 1
Saat x = 4: 2(4)-1 = 7
2x-1 pada [1, 4] menghasilkan [1, 7].

Range total: [0, 4] ∪ [1, 7] = [0, 7] ✅

Contoh 4: Evaluasi Piecewise

f(x) = { x+3, jika x < 0
        { x²,   jika 0 ≤ x < 2
        { 5,    jika x ≥ 2

f(-4) = -4 + 3 = -1 (x < 0, pakai aturan pertama)
f(0) = 0² = 0 (0 ≤ x < 2, pakai aturan kedua)
f(1,5) = (1,5)² = 2,25 (0 ≤ x < 2, pakai aturan kedua)
f(2) = 5 (x ≥ 2, pakai aturan ketiga)
f(100) = 5 (x ≥ 2, pakai aturan ketiga)
💡 Perhatikan batas interval: Tanda ≤ atau ≥ menentukan di mana batas interval "menempel". Di contoh di atas, x = 0 masuk ke aturan kedua (0 ≤ x < 2), bukan aturan pertama (x < 0). Pastikan tidak ada x yang masuk ke dua aturan sekaligus.

5. Komposisi Fungsi — Menggabungkan Dua Fungsi

Komposisi fungsi adalah operasi yang mengambil output dari satu fungsi dan menjadikannya input untuk fungsi lain.

(f ∘ g)(x) = f(g(x))

Dibaca: "f komposisi g" atau "f bundaran g" atau "f setelah g".

Urutan pengerjaan: hitung g(x) dulu, lalu masukkan hasilnya ke f.

Simbol ∘ disebut "bundaran" (circle).

Ilustrasi mesin:

Bayangkan dua mesin dalam pabrik. Mesin A memotong kayu, mesin B menghaluskan kayu. (g ∘ f)(kayu) artinya: masukkan kayu ke mesin f (potong), lalu hasilnya ke mesin g (haluskan). (f ∘ g)(kayu) artinya: masukkan ke mesin g dulu (haluskan kayu yang belum dipotong? aneh). Urutan sangat penting dan umumnya (f ∘ g) ≠ (g ∘ f).

Cara Menghitung Komposisi

f(x) = 2x + 1
g(x) = x² - 3

(f ∘ g)(x) = f(g(x))
= f(x² - 3)
= 2(x² - 3) + 1
= 2x² - 6 + 1
= 2x² - 5

(g ∘ f)(x) = g(f(x))
= g(2x + 1)
= (2x + 1)² - 3
= 4x² + 4x + 1 - 3
= 4x² + 4x - 2

Jelas: (f ∘ g) ≠ (g ∘ f). Hasilnya berbeda!

Komposisi dengan Fungsi yang Sama

f(x) = 3x - 2

(f ∘ f)(x) = f(f(x))
= f(3x - 2)
= 3(3x - 2) - 2
= 9x - 6 - 2
= 9x - 8

(f ∘ f)(4) = 9(4) - 8 = 36 - 8 = 28

Verifikasi manual: f(4) = 10, f(10) = 28. Cocok!

Domain Komposisi Fungsi

Ini adalah bagian yang sering terlewat dan bisa menjadi jebakan.

Domain (f ∘ g) = { x | x ∈ Dg DAN g(x) ∈ Df }

Syarat ganda:
1. x harus di domain g (agar g(x) terdefinisi).
2. g(x) harus di domain f (agar f(g(x)) terdefinisi).

Mengapa syarat kedua penting?

Meskipun x valid untuk g, hasil g(x) mungkin tidak valid untuk f. Contoh klasik: g(x) = x - 5 (domain ℝ), f(x) = √x (domain x ≥ 0). (f ∘ g)(x) = √(x - 5). Meskipun g menerima semua x, hasil g(x) harus ≥ 0 agar bisa diakarkan. Jadi domain (f ∘ g) = x ≥ 5.

f(x) = √x (domain: x ≥ 0)
g(x) = x - 4 (domain: ℝ)

(f ∘ g)(x) = √(x - 4)

Syarat: x - 4 ≥ 0 → x ≥ 4

Domain (f ∘ g): [4, ∞)

Meskipun g terdefinisi untuk semua x, hanya x ≥ 4 yang membuat g(x) masuk ke domain f.

Contoh Lebih Kompleks: Domain Komposisi

f(x) = 1/x (domain: x ≠ 0)
g(x) = x - 2 (domain: ℝ)

(f ∘ g)(x) = f(x-2) = 1/(x-2)

Syarat: g(x) ≠ 0 → x - 2 ≠ 0 → x ≠ 2

Domain (f ∘ g): (-∞, 2) ∪ (2, ∞)

Sifat-Sifat Komposisi Fungsi

1. Tidak komutatif: f ∘ g ≠ g ∘ f (umumnya).
2. Asosiatif: (f ∘ g) ∘ h = f ∘ (g ∘ h).
3. Identitas: f ∘ I = I ∘ f = f, di mana I(x) = x.

6. Fungsi Invers — Membalikkan Arah Fungsi

Invers fungsi (dilambangkan f⁻¹) adalah fungsi yang membalikkan efek dari f. Jika f memetakan a ke b, maka f⁻¹ memetakan b kembali ke a.

f(a) = b   ⇔   f⁻¹(b) = a

f⁻¹(f(x)) = x   dan   f(f⁻¹(x)) = x

Catatan penting: f⁻¹(x) BUKAN 1/f(x). Notasi pangkat -1 di sini adalah notasi invers, bukan kebalikan (resiprokal). Ini adalah sumber kebingungan paling umum.

Kapan Fungsi Punya Invers?

Tidak semua fungsi punya invers. Syaratnya:

Fungsi f memiliki invers jika dan hanya jika f bersifat bijektif (satu-satu dan onto).

Satu-satu (injektif): Setiap output berasal dari tepat satu input. f(a) = f(b) → a = b.

Onto (surjektif): Range = Kodomain (setiap elemen kodomain punya pasangan di domain).

Uji Garis Horizontal: Jika ada garis horizontal yang memotong grafik di lebih dari satu titik, fungsi tidak satu-satu → tidak punya invers (kecuali domainnya dibatasi).

Contoh klasik: f(x) = x² pada domain ℝ tidak punya invers, karena f(2) = 4 dan f(-2) = 4 (tidak satu-satu). Tapi jika domain dibatasi menjadi x ≥ 0, maka f(x) = x² punya invers f⁻¹(x) = √x.

Cara Mencari Invers Fungsi — Langkah Sistematis

  1. Tulis y = f(x).
  2. Tukar x dan y: x = f(y).
  3. Selesaikan persamaan untuk y (nyatakan y dalam x).
  4. Hasilnya adalah y = f⁻¹(x).
  5. Verifikasi: hitung f(f⁻¹(x)), harus = x.

Mengapa langkah "tukar x dan y" bekerja?

Invers membalikkan peran input dan output. Titik (a, b) di grafik f menjadi titik (b, a) di grafik f⁻¹. Menukar x dan y adalah cara aljabar untuk mewujudkan pertukaran ini.

Contoh 1: Fungsi Linear

f(x) = 3x - 7

y = 3x - 7
Tukar: x = 3y - 7
x + 7 = 3y
y = (x + 7)/3

f⁻¹(x) = (x + 7)/3

Verifikasi:
f(f⁻¹(x)) = 3((x+7)/3) - 7 = x + 7 - 7 = x ✅
f⁻¹(f(x)) = ((3x-7)+7)/3 = 3x/3 = x ✅

Contoh 2: Fungsi Rasional

f(x) = (2x+1)/(x-3), x ≠ 3

y = (2x+1)/(x-3)

Tukar: x = (2y+1)/(y-3)

x(y-3) = 2y+1
xy - 3x = 2y + 1
xy - 2y = 3x + 1
y(x-2) = 3x + 1
y = (3x+1)/(x-2), x ≠ 2

f⁻¹(x) = (3x+1)/(x-2), x ≠ 2

Verifikasi:
f(f⁻¹(x)) = f((3x+1)/(x-2))
= [2(3x+1)/(x-2) + 1] / [(3x+1)/(x-2) - 3]
= [(6x+2 + x-2)/(x-2)] / [(3x+1 - 3x+6)/(x-2)]
= [7x/(x-2)] / [7/(x-2)]
= 7x/7 = x ✅

Contoh 3: Fungsi Kuadrat dengan Domain Dibatasi

f(x) = x² + 4x + 3, dengan domain x ≥ -2

y = x² + 4x + 3
y = (x² + 4x + 4) - 4 + 3
y = (x+2)² - 1

Tukar: x = (y+2)² - 1
x + 1 = (y+2)²

Karena domain awal x ≥ -2, maka x+2 ≥ 0, sehingga y+2 juga ≥ 0.
y + 2 = √(x + 1)   (ambil akar positif)
y = √(x + 1) - 2

f⁻¹(x) = √(x + 1) - 2, dengan domain x ≥ -1 (dari x+1 ≥ 0)

Verifikasi:
f(f⁻¹(x)) = (√(x+1) - 2)² + 4(√(x+1) - 2) + 3
= (x+1) - 4√(x+1) + 4 + 4√(x+1) - 8 + 3
= x + 1 + 4 - 8 + 3 = x ✅

Sifat-Sifat Penting Invers Fungsi

1. (f⁻¹)⁻¹ = f (invers dari invers adalah fungsi aslinya).
2. f(f⁻¹(x)) = x dan f⁻¹(f(x)) = x (saling meniadakan).
3. (f ∘ g)⁻¹ = g⁻¹ ∘ f⁻¹ (invers komposisi = komposisi invers dengan urutan dibalik).
4. Grafik f⁻¹ adalah pencerminan grafik f terhadap garis y = x. Titik (a,b) pada f → (b,a) pada f⁻¹.

Mengapa (f ∘ g)⁻¹ = g⁻¹ ∘ f⁻¹?

Misalkan kita punya dua operasi: pakai baju dulu (g), baru pakai jaket (f). Untuk membalikkannya: lepas jaket dulu (f⁻¹), baru lepas baju (g⁻¹). Urutannya dibalik. Secara aljabar:

(f ∘ g) ∘ (g⁻¹ ∘ f⁻¹) = f ∘ (g ∘ g⁻¹) ∘ f⁻¹ = f ∘ I ∘ f⁻¹ = f ∘ f⁻¹ = I ✅

Hubungan Grafik Fungsi dan Inversnya

Grafik f dan f⁻¹ selalu simetris terhadap garis y = x.

Mengapa? Titik (a, b) pada grafik f berarti f(a) = b. Maka f⁻¹(b) = a, sehingga titik (b, a) pada grafik f⁻¹. Titik (a, b) dan (b, a) adalah pencerminan satu sama lain terhadap garis y = x (karena koordinatnya bertukar).

💡 Aplikasi: Jika kamu tahu grafik f, kamu bisa menggambar grafik f⁻¹ tanpa menghitung rumusnya: cukup cerminkan terhadap garis y = x.

📝 Ringkasan Cepat

  • Fungsi: aturan yang memasangkan setiap input ke tepat satu output.
  • Domain: semua input yang valid (hindari: penyebut 0, akar negatif, log non-positif).
  • Kodomain: himpunan output yang diizinkan (bagian dari definisi fungsi).
  • Range: output aktual yang dihasilkan. Range ⊆ Kodomain.
  • Piecewise: fungsi dengan aturan berbeda untuk interval berbeda.
  • Komposisi (f ∘ g)(x) = f(g(x)): kerjakan g dulu, baru f. Tidak komutatif.
  • Domain komposisi: x ∈ Dg dan g(x) ∈ Df.
  • Invers f⁻¹: membalikkan fungsi. Ada jika f satu-satu.
  • Mencari invers: y = f(x) → tukar x,y → selesaikan untuk y → verifikasi f(f⁻¹(x)) = x.
  • (f ∘ g)⁻¹ = g⁻¹ ∘ f⁻¹ (urutan dibalik).
  • Grafik f⁻¹: pencerminan grafik f terhadap y = x.

✨ Tips & Trik

  • Uji garis vertikal untuk mengecek apakah suatu grafik adalah fungsi. Uji garis horizontal untuk mengecek apakah fungsi punya invers.
  • Menentukan domain: tulis semua syarat (penyebut ≠ 0, radikan ≥ 0, numerus > 0). Iriskan semuanya. Jangan ada yang terlewat.
  • Menentukan range: gunakan metode "y = f(x), cari syarat x real". Ini metode paling sistematis dan jarang gagal.
  • Komposisi: urutan itu krusial. f ∘ g artinya g dulu, baru f. Jangan terbalik membaca (f ∘ g)(x) = f(g(x)), x masuk ke g, hasilnya ke f.
  • Verifikasi invers selalu. Hitung f(f⁻¹(x)). Kalau hasilnya bukan x, pasti ada yang salah dalam perhitunganmu.
  • Invers dari fungsi kuadrat perlu pembatasan domain. Tanpa pembatasan, fungsi kuadrat tidak satu-satu sehingga tidak punya invers.
  • Piecewise: perhatikan batas. Pastikan setiap x hanya masuk ke satu aturan. Cek nilai di titik batas untuk memastikan konsistensi.
  • Grafik adalah teman. Kalau bingung dengan sifat fungsi, gambar sketsa grafiknya. Visualisasi sering memberi insight yang tidak terlihat dari rumus.
