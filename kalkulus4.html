Part 4 • Ekstrem, Optimasi & Teorema

📖 Pendahuluan — Mencari Nilai Terbaik dan Terburuk

Di dunia nyata, kita terus-menerus mencari nilai optimal: bagaimana memaksimalkan keuntungan dengan biaya minimum, bagaimana mendesain kaleng dengan bahan paling sedikit, bagaimana rute terpendek antara dua titik.

Semua pertanyaan ini dijawab oleh teori optimasi dalam kalkulus. Dengan turunan, kita bisa menemukan titik-titik kritis di mana fungsi mencapai nilai maksimum atau minimum—baik secara lokal (di sekitar titik itu) maupun global (di seluruh domain).

Di bab ini, kita akan membangun teori dari dasar: dari perbedaan ekstrem lokal vs global, uji turunan pertama dan kedua, kecekungan dan titik belok, sampai teorema-teorema fundamental (Rolle, Teorema Nilai Rata-rata) yang menjadi tulang punggung seluruh kalkulus, dan Aturan L'Hôpital untuk menaklukkan limit bentuk tak tentu.

Tujuan Bab
  • Memahami perbedaan nilai maksimum/minimum lokal vs global secara mendalam.
  • Menguasai cara mencari titik kritis dan mengklasifikasikannya.
  • Menggunakan Uji Turunan Pertama dan Uji Turunan Kedua dengan tepat.
  • Memahami kecekungan (concavity) dan titik belok (inflection point).
  • Menyelesaikan berbagai jenis masalah optimasi di dunia nyata.
  • Memahami Teorema Rolle dan Teorema Nilai Rata-rata beserta bukti dan aplikasinya.
  • Menguasai Aturan L'Hôpital untuk berbagai bentuk tak tentu.

1. Nilai Maksimum dan Minimum — Dua Jenis "Puncak" dan "Lembah"

Ketika kita berbicara tentang "nilai terbesar" atau "nilai terkecil" suatu fungsi, kita harus berhati-hati: apakah kita berbicara tentang seluruh domain atau hanya di sekitar suatu titik? Inilah perbedaan antara global (absolute) dan lokal (relative).

Nilai maksimum global (absolute maximum):
f(c) adalah maksimum global jika f(c) ≥ f(x) untuk semua x di domain f.
Ini adalah "juara dunia"—tidak ada nilai yang lebih tinggi di mana pun.

Nilai minimum global (absolute minimum):
f(c) adalah minimum global jika f(c) ≤ f(x) untuk semua x di domain f.
Ini adalah "dasar jurang"—tidak ada nilai yang lebih rendah di mana pun.

Nilai maksimum lokal (relative maximum):
f(c) adalah maksimum lokal jika f(c) ≥ f(x) untuk semua x di sekitar c.
Ini adalah "juara kampung"—tertinggi di lingkungannya, tapi mungkin bukan yang tertinggi secara keseluruhan.

Nilai minimum lokal (relative minimum):
f(c) adalah minimum lokal jika f(c) ≤ f(x) untuk semua x di sekitar c.

Mengapa perbedaan ini penting?

Bayangkan sebuah fungsi seperti pegunungan dengan banyak puncak dan lembah.

Puncak Everest adalah maksimum global (titik tertinggi di seluruh pegunungan).
Puncak bukit kecil di kaki Everest adalah maksimum lokal (tertinggi di area situ, tapi lebih rendah dari Everest).
Lembah di antara dua bukit adalah minimum lokal.
Palung Mariana adalah minimum global.

Dalam matematika juga begitu: fungsi seperti y = x³ - 3x memiliki maksimum lokal di x = -1 dan minimum lokal di x = 1, tapi tidak memiliki maksimum atau minimum global karena fungsi ini menuju ±∞ saat x → ±∞.

1.1 Titik Kritis — Di Mana Ekstrem Bisa Terjadi?

Titik kritis dari fungsi f adalah titik x = c di mana:

1. f'(c) = 0 — gradien nol, garis singgung horizontal.
2. f'(c) tidak ada — fungsi tidak terdiferensialkan di situ (titik patah, sudut, garis vertikal).
3. c adalah titik ujung interval (jika domainnya interval tertutup).

Teorema Fermat: Jika f memiliki ekstrem lokal di c dan f'(c) ada, maka f'(c) = 0.

Akibat penting: Semua ekstrem lokal pasti terjadi di titik kritis.
Tapi tidak semua titik kritis adalah ekstrem—bisa jadi titik pelana (saddle point).

Mengapa f'(c) = 0 di titik ekstrem? — Bukti Teorema Fermat

Misalkan f memiliki maksimum lokal di c dan f'(c) ada.

Untuk x > c (di kanan c): [f(x) - f(c)]/(x-c) ≤ 0 (karena f(x) ≤ f(c) dan x-c > 0).
Limit kanan: f'(c⁺) ≤ 0.

Untuk x < c (di kiri c): [f(x) - f(c)]/(x-c) ≥ 0 (karena f(x) ≤ f(c) dan x-c < 0).
Limit kiri: f'(c⁻) ≥ 0.

Karena f'(c) ada, limit kiri = limit kanan = f'(c).
Maka 0 ≤ f'(c) ≤ 0 → f'(c) = 0. ✅

Intuisinya: di puncak bukit, permukaannya mendatar. Tidak mungkin miring ke atas atau ke bawah.

1.2 Contoh: Mencari Semua Titik Kritis

Soal 1: f(x) = x³ - 6x² + 9x + 1.

f'(x) = 3x² - 12x + 9 = 3(x² - 4x + 3) = 3(x-1)(x-3).
f'(x) = 0 → x = 1 atau x = 3.
f'(x) selalu ada (polinomial) → titik kritis: {1, 3}.

f(1) = 1 - 6 + 9 + 1 = 5. f(3) = 27 - 54 + 27 + 1 = 1.

Soal 2: f(x) = |x| (nilai mutlak).

Untuk x > 0: f'(x) = 1. Untuk x < 0: f'(x) = -1.
Di x = 0, limit kiri ≠ limit kanan → f'(0) tidak ada.
Titik kritis: {0} (karena f' tidak ada).

f(0) = 0, dan ini adalah minimum global. ✅

Soal 3: f(x) = x^(2/3) pada [-1, 8].

f'(x) = (2/3)x^(-1/3) = 2/(3∛x).
f'(x) tidak ada di x = 0 (penyebut nol).
f'(x) = 0 tidak mungkin karena pembilang konstan.
Titik ujung: x = -1 dan x = 8.
Titik kritis: {-1, 0, 8}. ✅

1.3 Teorema Nilai Ekstrem (Extreme Value Theorem — EVT)

Teorema Nilai Ekstrem:

Jika f kontinu pada interval tertutup [a, b], maka f pasti mencapai maksimum global dan minimum global di suatu titik dalam [a, b].

Mengapa butuh interval tertutup dan kontinu?

f(x) = x² pada (-2, 2) (interval terbuka): tidak ada maksimum global karena fungsi mendekati 4 tapi tidak pernah mencapainya di dalam interval.

f(x) = 1/x pada [-1, 1]: tidak kontinu di x = 0, tidak ada maks/min global.

f(x) = x² pada [-2, 2]: kontinu dan tertutup → minimum global di (0,0), maksimum global di (±2, 4). ✅

2. Uji Turunan Pertama — "Naik-Turun" Menentukan Jenis Ekstrem

Uji Turunan Pertama (First Derivative Test):

1. Cari semua titik kritis c (f'(c) = 0 atau f'(c) tidak ada).
2. Buat garis bilangan dan uji tanda f'(x) di setiap interval.
3. Tentukan jenis ekstrem dari perubahan tanda:

  • f' berubah dari positif (+) ke negatif (-) → maksimum lokal di c.
  • f' berubah dari negatif (-) ke positif (+) → minimum lokal di c.
  • f' tidak berubah tanda → bukan ekstrem (titik pelana/saddle point).

Mengapa perubahan tanda menentukan jenis ekstrem? — Intuisi Gerak

Bayangkan kamu sedang mendaki gunung.

f'(x) > 0 artinya kamu naik (elevasi bertambah).
f'(x) < 0 artinya kamu turun (elevasi berkurang).

Dari naik (+) ke turun (-): kamu mencapai puncak → maksimum lokal.
Dari turun (-) ke naik (+): kamu mencapai lembah → minimum lokal.
Tetap naik (+ ke +): kamu terus mendaki, belum mencapai puncak → bukan ekstrem.
Tetap turun (- ke -): kamu terus menurun → bukan ekstrem.

2.1 Contoh Lengkap dengan Garis Bilangan

f(x) = x³ - 3x² + 1. f'(x) = 3x² - 6x = 3x(x - 2).

Titik kritis: x = 0 dan x = 2.

Garis bilangan: ---0---2---

Interval (-∞, 0): Uji x = -1 → f'(-1) = 3(-1)(-3) = +9 > 0. Naik ↗
Interval (0, 2): Uji x = 1 → f'(1) = 3(1)(-1) = -3 < 0. Turun ↘
Interval (2, ∞): Uji x = 3 → f'(3) = 3(3)(1) = +9 > 0. Naik ↗

Di x = 0: naik → turun → maksimum lokal. f(0) = 1.
Di x = 2: turun → naik → minimum lokal. f(2) = 8 - 12 + 1 = -3. ✅

Verifikasi grafik: f naik dari -∞ ke x=0 (puncak), turun ke x=2 (lembah), naik ke ∞.

2.2 Contoh dengan Titik di Mana f' Tidak Ada

f(x) = x^(2/3). f'(x) = 2/(3∛x).

f'(x) tidak ada di x = 0. Tidak ada solusi f'(x) = 0.

Interval (-∞, 0): Uji x = -1 → f'(-1) = 2/(3(-1)) = -2/3 < 0. Turun ↘
Interval (0, ∞): Uji x = 1 → f'(1) = 2/3 > 0. Naik ↗

Di x = 0: turun → naik → minimum lokal (bahkan global). f(0) = 0. ✅

Grafiknya berbentuk "V" yang melengkung, dengan titik terdalam di (0,0).

2.3 Contoh Titik Kritis yang Bukan Ekstrem (Saddle Point)

f(x) = x³. f'(x) = 3x². Titik kritis: x = 0 (f'(0) = 0).

Interval (-∞, 0): Uji x = -1 → f'(-1) = 3 > 0. Naik ↗
Interval (0, ∞): Uji x = 1 → f'(1) = 3 > 0. Naik ↗

Tidak ada perubahan tanda → bukan ekstrem.

Grafiknya naik, mendatar sebentar di x=0 (titik belok), lalu naik lagi. Ini disebut titik pelana (saddle point) atau titik stasioner yang bukan ekstrem.

3. Uji Turunan Kedua — Lebih Cepat, Tapi Ada Batasnya

Uji Turunan Kedua (Second Derivative Test):

Misalkan f'(c) = 0 (titik stasioner) dan f''(c) ada:

• f''(c) > 0 → grafik cekung ke atas ∪ → minimum lokal.
• f''(c) < 0 → grafik cekung ke bawah ∩ → maksimum lokal.
• f''(c) = 0 → tidak dapat disimpulkan (perlu uji turunan pertama).

Catatan: Uji ini hanya bekerja untuk titik di mana f'(c) = 0. Tidak bisa dipakai untuk titik di mana f' tidak ada.

Mengapa f'' menentukan jenis ekstrem? — Intuisi Kecekungan

f'' mengukur kecekungan (curvature)—seberapa "melengkung" grafiknya.

f'' > 0 (cekung ke atas ∪): Grafik seperti mangkuk. Di titik terendah mangkuk, kamu berada di minimum. Kalau kamu sedikit bergeser ke kiri atau kanan, kamu naik.

f'' < 0 (cekung ke bawah ∩): Grafik seperti kubah. Di titik tertinggi kubah, kamu berada di maksimum. Kalau kamu sedikit bergeser, kamu turun.

f'' = 0: Kecekungan nol—grafik "lurus" di sekitar titik itu. Mungkin titik belok, mungkin juga ekstrem (seperti x⁴). Tidak bisa dipastikan.

3.1 Contoh Uji Turunan Kedua

f(x) = x³ - 6x² + 9x + 1. f'(x) = 3x² - 12x + 9. f''(x) = 6x - 12.

Titik kritis: f'(x) = 0 → 3(x-1)(x-3) = 0 → x = 1, 3.

Di x = 1: f''(1) = 6-12 = -6 < 0 → maksimum lokal. f(1) = 5.
Di x = 3: f''(3) = 18-12 = 6 > 0 → minimum lokal. f(3) = 1.

Hasilnya sama dengan uji turunan pertama, tapi lebih cepat! ✅

3.2 Kapan Uji Turunan Kedua Gagal?

Kasus 1: f''(c) = 0

f(x) = x³. f'(0) = 0. f''(0) = 0. Bukan ekstrem (titik belok).
f(x) = x⁴. f'(0) = 0. f''(0) = 0. Minimum lokal!

Keduanya punya f''(0) = 0, tapi hasilnya berbeda. Uji turunan kedua tidak bisa membedakan. Wajib pakai uji turunan pertama.

Kasus 2: f'(c) tidak ada

f(x) = |x|. f'(0) tidak ada. Uji turunan kedua tidak bisa dipakai.

Kesimpulan: Uji turunan kedua adalah jalan pintas yang cepat, tapi hanya berlaku untuk titik stasioner dengan f''(c) ≠ 0.

4. Kecekungan dan Titik Belok — "Kelengkungan" Grafik

Kecekungan (Concavity):

• f''(x) > 0 pada interval I → grafik cekung ke atas (concave up) di I.
  Bentuk: ∪ (seperti mangkuk, menampung air).

• f''(x) < 0 pada interval I → grafik cekung ke bawah (concave down) di I.
  Bentuk: ∩ (seperti kubah, menumpahkan air).

Titik Belok (Inflection Point):

Titik P(c, f(c)) adalah titik belok jika kecekungan berubah di c.
Syarat perlu: f''(c) = 0 atau f''(c) tidak ada.
Syarat cukup: f'' berubah tanda di sekitar c.

Mengapa titik belok penting?

Di titik belok, laju perubahan mencapai ekstrem lokal.

Dalam fisika: jika f(t) adalah posisi, maka f'(t) adalah kecepatan, f''(t) adalah percepatan. Titik belok adalah saat percepatan berubah tanda— dari mempercepat menjadi memperlambat, atau sebaliknya.

Dalam ekonomi: titik belok pada kurva pertumbuhan menunjukkan perubahan dari "pertumbuhan yang semakin cepat" ke "pertumbuhan yang melambat".

4.1 Contoh Mencari Kecekungan dan Titik Belok

f(x) = x³ - 3x. f'(x) = 3x² - 3. f''(x) = 6x.

f''(x) = 0 → x = 0.

Interval (-∞, 0): Uji x = -1 → f''(-1) = -6 < 0. Cekung ke bawah ∩
Interval (0, ∞): Uji x = 1 → f''(1) = 6 > 0. Cekung ke atas ∪

Kecekungan berubah di x = 0 → titik belok di (0, 0). ✅

Verifikasi: f'(0) = -3 (minimum lokal dari f'!). f''(0) = 0. Grafik berbentuk "S" — turun melandai di kiri, lalu naik di kanan.

4.2 Contoh Titik di Mana f'' Tidak Ada

f(x) = x^(1/3) (akar pangkat tiga).

f'(x) = 1/(3x^(2/3)). f''(x) = -2/(9x^(5/3)).

f''(x) tidak ada di x = 0.

Interval (-∞, 0): Uji x = -1 → f''(-1) = -2/(-9) > 0. ∪
Interval (0, ∞): Uji x = 1 → f''(1) = -2/9 < 0. ∩

Kecekungan berubah di x = 0 → titik belok di (0, 0). ✅

Grafiknya unik: dari cekung ke atas, melewati titik belok vertikal, lalu cekung ke bawah.

4.3 Titik Belok vs Titik Ekstrem — Tabel Perbandingan

| Aspek | Titik Ekstrem Lokal | Titik Belok | |---|---|---| | **Definisi** | f mencapai maksimum atau minimum di sekitar titik | Kecekungan berubah (∪↔∩) | | **f'(c)** | = 0 (atau tidak ada) | Bisa apa saja | | **f''(c)** | ≠ 0 biasanya (jika f'(c)=0) | = 0 atau tidak ada (biasanya) | | **Uji** | Uji turunan pertama atau kedua | Uji perubahan tanda f'' | | **Bentuk visual** | Puncak atau lembah | Gelombang "S" (cekung berubah) | | **Arti fisis** | Posisi tertinggi/terendah | Percepatan berubah tanda |

5. Optimasi — Mencari Nilai Terbaik dalam Kehidupan Nyata

Optimasi adalah puncak dari aplikasi turunan. Di sinilah kalkulus benar-benar "berbicara" dengan dunia nyata: mendesain produk dengan biaya minimum, memaksimalkan keuntungan, mencari rute terpendek, atau meminimalkan waste. Setiap masalah optimasi adalah teka-teki yang menunggu untuk dipecahkan.

Strategi Umum Menyelesaikan Masalah Optimasi:

Langkah 1 — Pahami: Baca soal dengan teliti. Apa yang dimaksimalkan/diminimalkan?
Langkah 2 — Gambar: Buat diagram/sketsa. Beri label variabel yang relevan.
Langkah 3 — Fungsi Objektif: Tulis fungsi yang akan dioptimasi (misal: luas, volume, biaya).
Langkah 4 — Kendala: Tulis persamaan yang menghubungkan variabel (constraint).
Langkah 5 — Satu Variabel: Gunakan kendala untuk mengubah fungsi objektif menjadi fungsi satu variabel.
Langkah 6 — Kritis: Cari turunan, tentukan titik kritis (f'(x) = 0 atau tidak ada).
Langkah 7 — Uji: Gunakan uji turunan pertama/kedua untuk memastikan maksimum/minimum.
Langkah 8 — Titik Ujung: Periksa nilai di ujung domain (jika domain terbatas).
Langkah 9 — Jawab: Tulis jawaban dalam kalimat lengkap dengan satuan.

5.1 Contoh 1: Pagar di Tepi Tembok — Memaksimalkan Luas

Soal: Seorang petani memiliki 120 meter pagar. Ia ingin membuat kandang persegi panjang yang satu sisinya berupa tembok (tidak perlu pagar). Berapa ukuran kandang agar luasnya maksimum?

Gambar: Persegi panjang dengan tembok di satu sisi panjang. Misal: x = lebar (dua sisi), y = panjang (satu sisi, karena sisi lainnya tembok).

Kendala: Total pagar = 2x + y = 120 → y = 120 - 2x.

Fungsi Objektif (Luas): A = x × y = x(120 - 2x) = 120x - 2x².

Domain: x > 0 dan y > 0 → x > 0 dan 120-2x > 0 → 0 < x < 60.

Titik Kritis: A'(x) = 120 - 4x = 0 → x = 30.

Uji: A''(x) = -4 < 0 → maksimum.

y = 120 - 2(30) = 60.

Ukuran optimal: lebar = 30 m, panjang = 60 m. Luas maks = 1800 m².

Verifikasi: x=20 → A=1600. x=40 → A=1600. x=30 → A=1800. Puncak di 30! ✅

5.2 Contoh 2: Kotak dari Karton — Memaksimalkan Volume

Soal: Selembar karton persegi berukuran 20 × 20 cm. Dari setiap sudut dipotong persegi kecil berukuran x × x, lalu dilipat membentuk kotak terbuka. Berapa x agar volume kotak maksimum?

Visualisasi: Setelah dipotong, alas kotak berukuran (20-2x) × (20-2x), dan tinggi kotak = x.

Volume: V(x) = x(20-2x)² = x(400 - 80x + 4x²) = 4x³ - 80x² + 400x.

Domain: 0 < x < 10 (agar 20-2x > 0).

Titik Kritis: V'(x) = 12x² - 160x + 400 = 0.
Bagi 4: 3x² - 40x + 100 = 0.
(3x - 10)(x - 10) = 0 → x = 10/3 ≈ 3,33 atau x = 10 (titik ujung).

Uji Turunan Kedua: V''(x) = 24x - 160.
V''(10/3) = 24(10/3) - 160 = 80 - 160 = -80 < 0 → maksimum.

Ukuran optimal: x = 10/3 ≈ 3,33 cm. Volume maks = V(10/3) ≈ 592,6 cm³.

Verifikasi: x=2 → V=512. x=4 → V=576. x=3,33 → V≈592,6. ✅

5.3 Contoh 3: Kaleng Silinder — Meminimalkan Bahan

Soal: Sebuah kaleng silinder harus memiliki volume 1000 cm³. Tentukan jari-jari dan tinggi yang meminimalkan luas permukaan (bahan yang digunakan untuk membuat kaleng, termasuk tutup).

Volume (kendala): V = πr²h = 1000 → h = 1000/(πr²).

Luas Permukaan (fungsi objektif): A = 2πr² + 2πrh (alas+tutup + selimut).

Substitusi h: A(r) = 2πr² + 2πr(1000/(πr²)) = 2πr² + 2000/r.

Domain: r > 0.

Titik Kritis: A'(r) = 4πr - 2000/r² = 0.
4πr = 2000/r² → r³ = 500/π → r = ∛(500/π) ≈ 5,42 cm.

Uji Turunan Kedua: A''(r) = 4π + 4000/r³ > 0 (selalu positif) → minimum.

h = 1000/(π(5,42)²) ≈ 10,84 cm.

Fakta menarik: h = 2r (tinggi = diameter). Kaleng paling efisien memiliki tinggi sama dengan diameter alasnya! ✅

5.4 Contoh 4: Jarak Terpendek ke Kurva

Soal: Cari titik pada parabola y = x² yang terdekat ke titik P(0, 3).

Fungsi Jarak: d = √[(x-0)² + (x²-3)²].

Minimalkan d² (lebih mudah, karena akar monoton):
D(x) = x² + (x²-3)² = x² + x⁴ - 6x² + 9 = x⁴ - 5x² + 9.

Domain: semua bilangan real (x ∈ ℝ).

Titik Kritis: D'(x) = 4x³ - 10x = 2x(2x² - 5) = 0.
x = 0 atau x = ±√(5/2) ≈ ±1,581.

Uji Turunan Kedua: D''(x) = 12x² - 10.
x = 0: D''(0) = -10 < 0 → maksimum lokal (titik terjauh).
x = ±√(5/2): D''(±√2,5) = 12(2,5) - 10 = 20 > 0 → minimum lokal.

D(±√2,5) = (2,5)² - 5(2,5) + 9 = 6,25 - 12,5 + 9 = 2,75.
Jarak minimum = √2,75 ≈ 1,658.

Titik terdekat: (√2,5, 2,5) dan (-√2,5, 2,5). ✅

6. Teorema Rolle — Di Antara Dua Akar, Ada Akar Turunan

Teorema Rolle (ditemukan Michel Rolle, 1691):

Jika f kontinu pada [a, b], terdiferensialkan pada (a, b), dan f(a) = f(b), maka ada c di (a, b) sedemikian sehingga f'(c) = 0.

Artinya: di antara dua titik dengan nilai fungsi yang sama, pasti ada setidaknya satu titik di mana garis singgungnya horizontal.

Intuisi Fisik — "Apa yang Naik Harus Turun":

Bayangkan kamu melempar bola vertikal ke atas. Bola mulai dari tanah (ketinggian 0), naik ke titik tertinggi, lalu turun kembali ke tanah (ketinggian 0 lagi).

f(t) = ketinggian bola. f(t₁) = 0, f(t₂) = 0 (kembali ke tanah).
Di antara t₁ dan t₂, ada momen di puncak tertinggi di mana kecepatan vertikal = 0.

f'(t_puncak) = 0. Inilah Teorema Rolle dalam aksi! ✅

6.1 Pembuktian Teorema Rolle

Kasus 1: f(x) = k (fungsi konstan) pada [a, b].
Maka f'(x) = 0 di mana-mana. Setiap c di (a, b) memenuhi. ✅

Kasus 2: f(x) > f(a) untuk suatu x di (a, b).
Dengan Teorema Nilai Ekstrem, f mencapai maksimum global di suatu c ∈ (a, b) (karena f(a)=f(b) dan ada nilai yang lebih besar, maksimum tidak mungkin di ujung).
Di c, f memiliki maksimum lokal → dengan Teorema Fermat: f'(c) = 0. ✅

Kasus 3: f(x) < f(a) untuk suatu x di (a, b).
Argumen serupa, f mencapai minimum global di suatu c ∈ (a, b) → f'(c) = 0. ✅

6.2 Aplikasi Teorema Rolle — Menjamin Keberadaan Akar Turunan

Soal: f(x) = x³ - 3x pada [-√3, √3].

f(-√3) = -3√3 + 3√3 = 0. f(√3) = 3√3 - 3√3 = 0.
f(a) = f(b) = 0 → Rolle berlaku!

f'(x) = 3x² - 3 = 3(x² - 1) = 0 → x = -1 atau 1.
Keduanya di dalam (-√3, √3). Jadi ada dua nilai c: c = -1 dan c = 1.

Makna: Di antara dua akar fungsi (x = ±√3), ada dua titik di mana garis singgung horizontal (x = ±1). ✅

6.3 Kenapa Semua Syarat Rolle Diperlukan?

Tanpa Kontinuitas: f(x) = x pada [0, 1] dengan f(1) diubah jadi f(1) = 0. f(0) = 0, f(1) = 0, tapi f tidak kontinu di x=1. Tidak ada c dengan f'(c) = 0 di (0,1).

Tanpa Diferensiabilitas: f(x) = |x| pada [-1, 1]. f(-1) = f(1) = 1. Tapi f tidak terdiferensialkan di x=0 (titik patah). f'(x) tidak pernah 0.

Tanpa f(a) = f(b): f(x) = x pada [0, 1]. f(0) = 0, f(1) = 1 (tidak sama). f'(x) = 1 (tidak pernah 0). ❌

7. Teorema Nilai Rata-rata (MVT) — Jantung Kalkulus

Teorema Nilai Rata-rata (Mean Value Theorem — MVT):

Jika f kontinu pada [a, b] dan terdiferensialkan pada (a, b), maka ada c di (a, b) sedemikian sehingga:

f'(c) = [f(b) - f(a)] / (b - a)

Artinya: ada titik di mana laju perubahan sesaat = laju perubahan rata-rata.

Intuisi — "Anda Pasti Pernah Tepat di Kecepatan Rata-Rata":

Kamu berkendara dari Jakarta ke Bandung (150 km) dalam 2 jam. Kecepatan rata-ratamu = 75 km/jam.

MVT menjamin: pasti ada setidaknya satu momen di perjalanan itu di mana speedometer menunjukkan tepat 75 km/jam.

Kenapa? Karena kamu tidak mungkin selalu di atas 75 km/jam (lalu bagaimana bisa rata-ratanya 75?) atau selalu di bawah 75 km/jam. Harus ada saat di mana kamu melintasi batas 75 km/jam.

7.1 Pembuktian MVT dari Teorema Rolle

Definisikan fungsi bantu:

g(x) = f(x) - [f(a) + ((f(b)-f(a))/(b-a))(x-a)]

g(x) adalah selisih vertikal antara f(x) dan garis secant dari (a,f(a)) ke (b,f(b)).

g(a) = f(a) - f(a) = 0. g(b) = f(b) - f(b) = 0.
g kontinu dan terdiferensialkan (karena f juga).

Dengan Teorema Rolle: ada c di (a,b) dengan g'(c) = 0.

g'(x) = f'(x) - (f(b)-f(a))/(b-a).
g'(c) = 0 → f'(c) = (f(b)-f(a))/(b-a). ✅

7.2 Contoh MVT — Mencari Nilai c

f(x) = x² pada [1, 4].

f(1) = 1, f(4) = 16. Laju rata-rata = (16-1)/(4-1) = 15/3 = 5.

f'(x) = 2x. MVT: f'(c) = 5 → 2c = 5 → c = 2,5.

c = 2,5 ada di dalam (1, 4). ✅

Di x = 2,5, laju sesaat = 5, persis sama dengan laju rata-rata di interval [1,4].

7.3 Akibat-Akibat Penting MVT — Fondasi Kalkulus

Akibat 1: Jika f'(x) = 0 untuk semua x di interval I, maka f konstan di I.

Akibat 2: Jika f'(x) = g'(x) untuk semua x di I, maka f(x) = g(x) + C.

Akibat 3: Jika f'(x) > 0 di I, maka f naik tegas di I.
Jika f'(x) < 0 di I, maka f turun tegas di I.

Akibat 4 (Teorema Rolle sebagai kasus khusus):
Jika f(a) = f(b), maka f'(c) = (f(b)-f(a))/(b-a) = 0. Inilah Rolle!

Mengapa MVT disebut "jantung kalkulus"? Karena hampir semua teorema dalam kalkulus (termasuk Teorema Dasar Kalkulus, Aturan L'Hôpital, deret Taylor) bergantung pada MVT. Tanpa MVT, kalkulus runtuh.

8. Aturan L'Hôpital — Senjata Pamungkas untuk Limit Sulit

Aturan L'Hôpital (dipopulerkan oleh Marquis de L'Hôpital, 1696, tapi sebenarnya ditemukan oleh Johann Bernoulli):

Jika lim[x→c] f(x)/g(x) berbentuk 0/0 atau ∞/∞, maka:

lim[x→c] f(x)/g(x) = lim[x→c] f'(x)/g'(x)

Asalkan limit di ruas kanan ada (atau menuju ±∞).

Aturan ini juga berlaku untuk x → ∞, x → -∞, dan limit sepihak.

Mengapa ini berhasil? — Intuisi dari MVT

Untuk kasus 0/0 dengan f(c) = g(c) = 0:

f(x)/g(x) = [f(x)-f(c)]/[g(x)-g(c)]
= ([f(x)-f(c)]/(x-c)) / ([g(x)-g(c)]/(x-c))

Saat x → c, pembilang → f'(c), penyebut → g'(c) (definisi turunan!).

Jadi limitnya = f'(c)/g'(c). ✅

Untuk kasus ∞/∞ atau x → ∞: pembuktiannya lebih teknis, tapi intuisinya sama: L'Hôpital "mempercepat" konvergensi dengan membandingkan laju perubahan, bukan nilai absolutnya.

8.1 Contoh Dasar L'Hôpital

1. lim[x→0] sin x / x (bentuk 0/0)
= lim[x→0] cos x / 1 = 1. ✅

2. lim[x→0] (eˣ - 1)/x (bentuk 0/0)
= lim[x→0] eˣ / 1 = 1. ✅

3. lim[x→0] (1 - cos x)/x² (bentuk 0/0, butuh 2 kali L'Hôpital)
= lim[x→0] sin x / (2x) (masih 0/0!)
= lim[x→0] cos x / 2 = 1/2. ✅

4. lim[x→∞] ln x / x (bentuk ∞/∞)
= lim[x→∞] (1/x) / 1 = 0. ✅

5. lim[x→∞] eˣ / x² (bentuk ∞/∞, butuh 2 kali)
= lim[x→∞] eˣ / (2x) (masih ∞/∞!)
= lim[x→∞] eˣ / 2 = ∞. ✅

8.2 Bentuk Tak Tentu yang Harus Diubah Dulu

L'Hôpital hanya bekerja untuk 0/0 dan ∞/∞. Bentuk lain harus dimanipulasi dulu.

0 × ∞: Ubah jadi 0/0 atau ∞/∞ dengan memindahkan salah satu faktor ke penyebut.

∞ - ∞: Samakan penyebut atau faktorkan.

1^∞, 0⁰, ∞⁰: Gunakan logaritma natural: ln(y) = ... lalu cari limit ln(y).
6. lim[x→0⁺] x ln x (bentuk 0 × (-∞))
Ubah: x ln x = ln x / (1/x) → bentuk -∞/∞.
= lim[x→0⁺] (1/x) / (-1/x²) = lim[x→0⁺] (-x) = 0. ✅

7. lim[x→∞] (1 + 1/x)ˣ (bentuk 1^∞ — trik: gunakan ln)
Misal y = (1 + 1/x)ˣ. ln y = x ln(1 + 1/x).

lim[x→∞] ln y = lim[x→∞] ln(1+1/x) / (1/x) (bentuk 0/0)
= lim[x→∞] [(-1/x²)/(1+1/x)] / (-1/x²)
= lim[x→∞] 1/(1+1/x) = 1.

Maka lim[x→∞] y = e¹ = e. ✅

8.3 Kapan L'Hôpital Tidak Boleh Dipakai?

⚠️ JANGAN pakai L'Hôpital jika:

❌ Limit bukan bentuk 0/0 atau ∞/∞. Cek dulu!
❌ Limit f'(x)/g'(x) tidak ada (bukan berarti limit aslinya tidak ada).
❌ Terjebak dalam siklus (misal: lim[x→∞] x/√(x²+1) — lebih baik pakai aljabar).

Contoh penyalahgunaan:

lim[x→0] (x + sin x)/x (bentuk 0/0) → L'Hôpital → lim(1+cos x)/1 = 2. ✅

Tapi lim[x→0] (x² sin(1/x))/sin x: L'Hôpital membuatnya makin rumit! Lebih baik pakai teorema squeeze.

📝 Ringkasan Cepat Kalkulus 4

  • Titik kritis: f'(c)=0, f'(c) tidak ada, atau titik ujung. Kandidat ekstrem.
  • Uji Turunan Pertama: + ke - → maks lokal. - ke + → min lokal. Tidak berubah → bukan.
  • Uji Turunan Kedua: f''(c)>0 → min. f''(c)<0 → maks. f''(c)=0 → tidak tahu (pakai uji pertama).
  • Kecekungan: f''>0 → ∪. f''<0 → ∩. Titik belok: kecekungan berubah (f''=0 atau tidak ada + perubahan tanda).
  • Optimasi: 9 langkah: pahami → gambar → objektif → kendala → satu variabel → kritis → uji → ujung → jawab.
  • Teorema Rolle: f(a)=f(b) → ada c dengan f'(c)=0. Perlu kontinu + terdiferensialkan.
  • Teorema Nilai Rata-rata: f'(c) = (f(b)-f(a))/(b-a). Laju sesaat = laju rata-rata. Jantung kalkulus!
  • Aturan L'Hôpital: 0/0 atau ∞/∞ → turunkan pembilang dan penyebut. Bisa berulang. Bentuk lain harus diubah dulu.

✨ Tips & Trik

  • Selalu periksa titik ujung dalam optimasi! Maksimum/minimum global bisa terjadi di ujung interval, bukan di titik stasioner. Ini kesalahan yang sangat umum.
  • f''(c) = 0 → jangan memaksakan kesimpulan. Langsung pakai uji turunan pertama. Contoh: x³ dan x⁴ sama-sama f''(0)=0, tapi hasilnya berbeda.
  • L'Hôpital bisa dipakai berulang. Kalau setelah satu kali turunan masih 0/0 atau ∞/∞, turunkan lagi. Tapi jangan lupa cek bentuknya setiap kali!
  • Optimasi: gambar dulu! Visualisasi masalah membantu menemukan fungsi objektif dan kendala. Sketsa sederhana sudah cukup.
  • MVT adalah "mesin" di balik banyak teorema. Kalau kamu paham MVT, kamu bisa menurunkan Rolle, sifat naik/turun, dan bahkan Teorema Dasar Kalkulus.
  • Bentuk 1^∞, 0⁰, ∞⁰: logaritma natural adalah kuncinya. Ambil ln dari kedua sisi, cari limit ln(y), lalu eksponensialkan hasilnya.
  • Verifikasi selalu! Setelah optimasi, cek nilai di sekitar titik kritis. Kalau bisa, bandingkan dengan intuisi atau perhitungan kasar.
